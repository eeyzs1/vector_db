# 性能测试指南

## 概述

本指南提供了如何使用项目中的性能测试工具来评估7VecDB与其他向量数据库的性能差异。测试工具包括基础性能测试、并发性能测试和结果分析。

## 测试环境

### 硬件环境
- **CPU**: AMD Ryzen 9 9955HX3D (8核)
- **内存**: 32GB DDR4
- **存储**: SSD 1TB
- **网络**: 千兆以太网

### 软件环境
- **操作系统**: Windows 11
- **Python**: 3.9+
- **依赖**: 见项目根目录的 `requirements.txt`

## 测试工具

### 1. 基础性能测试
- **文件**: `performance_test.py`
- **功能**: 测试插入性能、搜索性能、索引构建性能

### 2. 并发性能测试
- **文件**: `load_test.py` (基于Locust)
- **功能**: 测试并发用户下的系统性能

### 3. 综合性能测试
- **文件**: `run_performance_tests.py`
- **功能**: 协调不同数据库后端的性能测试

### 4. 结果分析工具
- **文件**: `analyze_performance_results.py`
- **功能**: 分析测试结果并生成可视化图表

## 测试数据

### 数据集
1. **SIFT500K** - 50万向量，128维
2. **GIST500K** - 50万向量，960维
3. **Wikipedia Text** - 5万文本片段，384维
4. **ImageNet Subset** - 5万图像特征，512维

### 测试规模
- **插入测试**: 5K, 50K, 500K向量
- **搜索测试**: k=1, 5, 10, 50
- **并发测试**: 1, 5, 50, 250并发用户

## 运行测试

### 1. 安装依赖

```powershell
# 安装项目依赖
pip install -r requirements.txt

# 安装额外的测试依赖
pip install psutil matplotlib locust
```

### 2. 运行综合性能测试

```powershell
# 运行所有性能测试
python tests/run_performance_tests.py
```

### 3. 运行并发性能测试

```powershell
# 启动7VecDB服务
uvicorn api.main:app --host 0.0.0.0 --port 8001

# 在另一个终端运行Locust测试
locust -f tests/load_test.py --host http://localhost:8001 --run-time 60s --users 50 --spawn-rate 5 --headless --csv tests/loadtest_results/test_results
```

### 4. 分析测试结果

```powershell
# 分析性能测试结果
python tests/analyze_performance_results.py
```

## 测试结果

### 结果目录
- **`tests/performance_results/`**: 性能测试结果
  - `*.json`: 原始测试结果
  - `performance_report.json`: 综合性能报告
  - `performance_summary.json`: 性能摘要
  - `*.png`: 性能图表

- **`tests/loadtest_results/`**: 并发测试结果
  - `*.csv`: Locust测试结果

### 性能指标

| 指标 | 单位 | 说明 |
|------|------|------|
| 插入吞吐量 | vectors/second | 每秒插入的向量数量 |
| 搜索延迟 | ms | 单条搜索操作的平均响应时间 |
| QPS | queries/second | 每秒可处理的查询数量 |
| 索引构建时间 | seconds | 构建索引所需的时间 |
| 内存使用 | MB | 运行时内存占用 |

## 测试场景

### 1. 基础性能测试
- **插入性能**: 测试不同规模向量的插入速度
- **搜索性能**: 测试不同k值下的搜索速度和准确率
- **索引构建**: 测试索引构建时间和资源消耗

### 2. 高级性能测试
- **并发性能**: 测试多用户同时操作的系统性能
- **混合查询**: 测试结合元数据过滤的向量搜索
- **持久性测试**: 测试重启服务后的数据恢复性能
- **扩展性测试**: 测试不同数据规模下的性能变化

## 注意事项

1. **测试公平性**: 确保所有数据库使用相同的硬件和软件环境
2. **数据一致性**: 使用相同的测试数据和嵌入模型
3. **参数调优**: 每个数据库使用最优配置参数
4. **结果可重现**: 记录所有测试参数和环境配置
5. **统计显著性**: 多次测试取平均值，确保结果可靠

## 扩展测试

### 特殊场景测试
- **高维向量**: 测试1024维以上向量的性能
- **稀疏向量**: 测试稀疏向量的处理性能
- **实时更新**: 测试实时数据更新场景
- **混合检索**: 测试文本+向量混合检索

### 长期稳定性测试
- **24小时持续运行测试**
- **数据增长测试**
- **故障恢复测试**

通过本测试工具，您可以全面评估7VecDB的性能表现，识别其优势和改进空间，为用户提供客观的性能参考。同时，测试结果也可以为项目的持续优化提供数据支持。
